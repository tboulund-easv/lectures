# The Problem with Metrics

- Vanity metrics (e.g., lines of code)  
- Output ≠ outcome  
- Risk of gaming metrics

# What Makes a Good Metric?

::left::

- Actionable  
- Easy to understand  

::right::

- Hard to game  
- Balanced

# Introducing DORA

- Industry standard (Google research)  
- Focus on delivery performance  
- Links engineering to business outcomes  

---
layout: default
---

# The Four DORA Metrics

1. 🚀 Deployment Frequency  
2. ⏱ Lead Time for Changes  
3. ❌ Change Failure Rate  
4. 🔧 Time to Restore Service (MTTR)  

---
layout: image-left
image: https://source.unsplash.com/featured/?pipeline,software
---

# DORA in Practice

- Fast + stable = high performance  
- Measure trends over time  
- Avoid optimizing one metric alone  

---
layout: center
---

# Example Interpretation

- ✅ High frequency + low failures = ideal  
- ⚠️ Fast lead time + high failures = risky  
- 🐢 Slow + stable = improvement opportunity

# Limitations of DORA

::left::

- Focus on system outcomes  
- Ignores developer experience  

::right::

- Can create pressure  
- Risk of misuse

# Introducing SPACE

- Developed by GitHub

- Includes human factors

# SPACE Dimensions

- 😊 Satisfaction & Well-being

- 🌊 Efficiency & Flow

# Why SPACE Matters

- Productivity is multidimensional  
- Avoids over-reliance on activity  
- Supports healthier teams

# DORA vs SPACE

::left::

## DORA
- System performance  
- Delivery-focused  

::right::

## SPACE
- Human + team dynamics  
- Holistic productivity

# How to Use Both

- Use DORA for delivery outcomes  
- Use SPACE for team health  
- Combine quantitative + qualitative data  

---
layout: default
---

# Practical Implementation

- Start small (2–4 metrics)  
- Track trends, not snapshots  
- Use metrics for learning

# Common Mistakes

- Measuring too much  
- Using metrics for control  
- Ignoring context  
- Focusing only on output

# Key Takeaways

- No single metric tells the full story  
- DORA = delivery performance  
- SPACE = developer experience  
- Measure to improve, not judge
